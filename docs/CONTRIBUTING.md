# Guia de Contribuição de Documentação

## Padrão editorial
- Escreva em Português-BR, curto e direto.
- Estruture em: **Contexto**, **Decisão/Padrão**, **Exemplos**, **Checklist**.
- Use termos consistentes com `docs/glossario.md`.
- Marque incertezas com **A definir**.

## Definition of Done (DoD) para docs
- [ ] Documento atualizado no domínio correto
- [ ] Fluxo impactado descrito ponta a ponta
- [ ] Segurança/impacto operacional considerados
- [ ] Links internos revisados
- [ ] Checklist de validação incluído

## Versionamento de docs
- Documentação deve acompanhar mudanças de código no mesmo PR.
- Mudanças arquiteturais devem atualizar `architecture/` e `roadmap/`.
- Mudanças de API devem atualizar `api/`.
