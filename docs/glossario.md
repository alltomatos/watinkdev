# Glossário Técnico

Termos oficiais usados na documentação do projeto.

> Se um termo não tiver definição validada pelo time, manter como **A definir**.

## A

- **API**: Interface de programação para comunicação entre sistemas.
- **A definir**: marcador obrigatório para conteúdo ainda não formalizado.

## B

- **Backend**: camada de serviços, regras de negócio e acesso a dados.

## E

- **Engine**: componente responsável por integração de mensageria e processamento associado.

## F

- **Frontend**: camada de interface com o usuário.

## M

- **Multi-tenant**: arquitetura onde múltiplos clientes compartilham a mesma aplicação com isolamento lógico.

## P

- **Plugin**: módulo complementar que adiciona funcionalidade específica ao sistema.

## R

- **RBAC**: controle de acesso baseado em papéis.
- **Runbook**: procedimento operacional padronizado para execução técnica.

## S

- **SaaS**: modelo de entrega de software como serviço.

## T

- **Tenant**: unidade de cliente/organização dentro de contexto multi-tenant.

## Checklist prático

- [ ] Termos críticos dos domínios foram incluídos.
- [ ] Não há sinônimos conflitantes.
- [ ] Definições ambíguas estão marcadas como **A definir**.
- [ ] Termos novos foram referenciados nos documentos de domínio.
